RapidIdentity Disk Expansion Cheat Sheet

Pre‑Check Partition Physical Volume Volume Group Logical Volume Grow FS Validation Troubleshooting

0. Pre‑Check

  1. Ask the customer to add space to the VM – either increase the size of the primary disk or attach a new one.
  2. Confirm free space and mount points: 
    df -h
    If the root filesystem is nearly full, clean up logs first so the appliance does not flip to read‑only during the resize.
  3. If unsure which device changed, list block devices: 
    lsblk

1. Create a Partition

  1. Identify the correct disk (usually /dev/sda if the original disk was extended, or /dev/sdb for a second disk): 
    fdisk -l
  2. Start fdisk on that device and add a primary partition of type 8e (Linux LVM): 
    fdisk /dev/sdX   # replace X with a or b
p                # print table
n                # new partition
p                # primary
     # use next free partition number
           # default first cylinder
           # default last cylinder
t                # change type

8e               # Linux LVM
w                # write table and quit
  3. Reboot so the kernel detects the new partition table.

2. Add Physical Volume

  1. Create the physical volume on the new partition: 
    pvcreate /dev/sdXX
  2. Verify: 
    pvdisplay

3. Extend Volume Group

  1. Find the volume group name (CentOS default is centos): 
    vgdisplay
  2. Add the new PV: 
    vgextend centos /dev/sdXX
  3. Confirm free space now exists in the VG.

4. Resize Logical Volume

  1. Show current LV size: 
    lvdisplay
  2. Grow the root LV using all free extents: 
    lvextend --extents +100%FREE /dev/centos/root
  3. Verify the LV reflects the new size.

5. Grow Filesystem

  1. Expand the XFS root filesystem online: 
    xfs_growfs /dev/centos/root
  2. Run df -h again to confirm the new size is available.

6. Validation

  1. Check RapidIdentity UI loads without errors.
  2. Monitor /var/log/messages for LVM or XFS warnings.
  3. Reboot once more and verify the expanded filesystem persists.

7. Troubleshooting & Quick Commands

Common Issues

  • Partition table not detected – reboot after writing changes with fdisk.
  • pvcreate fails – confirm the partition type is 8e.
  • lvextend says no free extents – ensure vgextend ran on the correct VG.
  • xfs_growfs: cannot enlarge – LV did not grow; re‑check lvdisplay.

Cheat‑Sheet Commands

# View space
 df -h

# List block devices
 lsblk

# Create PV, extend VG, extend LV
 pvcreate /dev/sdXX
 vgextend centos /dev/sdXX
 lvextend --extents +100%FREE /dev/centos/root

# Grow filesystem
 xfs_growfs /dev/centos/root

# Verify
 df -h
