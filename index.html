RapidIdentity Disk Expansion Cheat Sheet

Pre-Check Partition Physical Volume Volume Group Logical Volume Grow FS Validation Troubleshooting

0. Pre‑Check

  1. Have the virtualization admin extend the virtual disk (or attach a new one). A good rule of thumb is to double the current size unless the disk is already hundreds of GB.
  2. Review RapidIdentity Security → Retention Policies. Switch any non‑essential logs from Always Retain to Inherit (default 7 days) so the appliance does not quickly outgrow the new space.
  3. Confirm you are on the correct appliance (e.g., DB node) by checking its IP with: 
    ifconfig | grep inet
  4. Check current usage and mounts: 
    df -h
  5. List block devices and sizes (verify the hypervisor change is visible): 
    lsblk

1. Create a Partition

  1. If lsblk does not yet reflect the larger size, rescan the SCSI bus (no reboot needed): 
    # replace sda with the correct device if different
echo 1 > /sys/class/block/sda/device/rescan
    Re‑run lsblk to confirm the new size is detected.
  2. Identify the target disk (usually /dev/sda). Launch fdisk and add a new primary partition of type 8e (Linux LVM): 
    fdisk /dev/sdX   # replace X with a or b
n  # new partition
p  # primary
     # next free partition number
           # default first sector
           # default last sector (use full space)
t  # change type

8e # Linux LVM
w  # write & quit
    If fdisk warns "no free sectors available," the hypervisor resize was not detected; rescan or reboot and try again.
  3. Run partprobe to notify the kernel of the new table (older kernels may still require a reboot).

2. Add Physical Volume

  1. Create the PV: 
    pvcreate /dev/sdXX
  2. Verify with: 
    pvdisplay

3. Extend Volume Group

  1. Find the VG (CentOS default is centos): 
    vgdisplay
  2. Add the new PV: 
    vgextend centos /dev/sdXX

4. Resize Logical Volume

  1. Display current LVs: 
    lvdisplay
  2. Grow the root LV using all free space: 
    lvextend --extents +100%FREE /dev/centos/root

5. Grow Filesystem

  1. Expand XFS online: 
    xfs_growfs /dev/centos/root
  2. Confirm with df -h.

6. Validation

  1. Load RapidIdentity UI – ensure normal login and workflows.
  2. Tail /var/log/messages for LVM/XFS warnings.
  3. Schedule a reboot during a maintenance window and re‑run df -h afterwards.

7. Troubleshooting & Quick Commands

Common Issues

  • Hypervisor resized but OS does not see it – run echo 1 > /sys/class/block/sdX/device/rescan or reboot.
  • fdisk says "No free sectors available" – resize not detected (see above rescan).
  • pvcreate fails – ensure the partition type is 8e.
  • lvextend: No free extents – confirm vgextend was successful.
  • xfs_growfs cannot enlarge – LV did not grow.

Cheat‑Sheet Commands

# Verify appliance & space
ifconfig | grep inet
df -h
lsblk

# Rescan after hypervisor resize
echo 1 > /sys/class/block/sda/device/rescan

# Partition & LVM
fdisk /dev/sda
partprobe
pvcreate /dev/sda4
vgextend centos /dev/sda4
lvextend --extents +100%FREE /dev/centos/root
xfs_growfs /dev/centos/root

# Check results
df -h
